## 🎬 任务执行流程

```mermaid
stateDiagram-v2
    [*] --> Idle: 等待用户消息
    Idle --> Processing: 收到消息
    Processing --> DST_Update: 创建 UserUtter
    DST_Update --> DPL_Generate: 更新 DST
    DPL_Generate --> Action_Build: 生成动作名称
    Action_Build --> Action_Execute: 构建 Action
    Action_Execute --> Response: 执行 Action
    Response --> Check_Form: 发送响应
    Check_Form --> DPL_Generate: 继续循环
    Check_Form --> Save: BotUtter 结束
    Save --> Idle: 保存 DST
    
    note right of DPL_Generate
        如果 DPL 失败
        降级到 Selector
    end note
    
    note right of Action_Execute
        支持多种类型:
        API 调用、LLM 生成、
        RAG 检索等
    end note
```

## 📊 性能特征

| 组件 | 时间复杂度 | 空间复杂度 | 优化建议 |
|------|-----------|-----------|---------|
| **DST update** | O(1) | O(n) | n=对话轮数 |
| **DPL generate** | O(m) | O(1) | m=策略数量 |
| **Action run** | O(1) | O(k) | k=LLM 响应长度 |
| **Store save** | O(1) | O(n) | MemoryStore |
| **Store retrieve** | O(1) | O(n) | 哈希查找 |
